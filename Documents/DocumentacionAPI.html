TICKCHECKTING API v1.0.0

A continuacion se presenta la documentacion oficial para el uso correcto de la API de TICKCHECKTING, la cual esta desarrollada en Spring Tools Suite en lenguaje Java.

API Host

Todo tipo de peticiones se deben hacer con la siguiente URL:

https://projectncapas-c3f52370649e.herokuapp.com/

Interactuar con la API

Si quieres probar nuestra API de una manera rápida te recomendamos el uso de Postman o Insomnia. Postman es una extensión de Google Chrome que permite interactuar con HTTP API’s de forma sencilla a través de una interfaz amigable para construir peticiones y obtener respuestas e Insomnia es un programa más completo que también permite interactuar con HTTP API’s y también crearlas.

Autentificación

Las autenticaciones se harán por medio de HTTP Basic Auth que al mismo tiempo de comprobarán con un token. Por medio de este token autenticado es que se podrán hacer requests a la API, no se podrán hacer en caso de no tenerlo. También el token se obtiene será de forma temporal por lo que tendrás un tiempo limite para poder interactuar y hacer peticiones a la API, esto es para principalmente evitar que un usuario no deseado pueda obtener el token.

Errores comunes

CODIGO ERROR
200 OK. Todo ha funcionado, ningún problema.
400 Bad Request. El request no ha sido aceptado, generalmente por un parámetro que está vacío.
401 Unauthorized. El token es inválido.
402 Request Failed. Los parámetros han sido aceptados, pero el request falló.
403 Forbidden. El token dado no tiene permisos para hacer este request.
404 Not Found. El recurso que el request ha pedido no existe.
409 Conflict. El request tiene problemas con otro request (tal vez por el uso de un mismo token)
429 Too Many Requests. Muchos request para que la API pueda procesarlos correctamente. Si este es el caso recomendamos deshacer tus requests o en su defecto dejar de hacer requests.
500, 502, 503, 504 Errores del servidor. Si este es el caso, es por errores de parte del equipo de LookingPlace, si experimenta con alguno de ellos puede reportar el error.

Usuarios

POST

Crear un nuevo usuario

La ruta para realizar la peticion es la siguiente:
POST https://projectncapas-c3f52370649e.herokuapp.com/API/v1/tickcheck/auth/singup

La creacion de usuarios es una de las pocas rutas de la Api que no necesita un token para poder ser utilizada. Los parametros que necesita la peticion son los siguientes:

  • Username: sera el nombre con el cual sera creada la cuenta, este debera de ser unico, caso contrario genera error.
  • Email: correo electronico del usario, este debera de ser unico.
  • Password: contrasena de minimo 8 caracterres los cuales debera de contar almenos una mayuscula, una minuscula y un caracter especial.

Un ejemplo del cuerpo de la peticion es el siguiente:

{
	"username":"erick",
	"email":"erick@gmail.com",
	"password": "1234567tE$"
}

La respuesta a la peticion anterios es la siguiente:

{
	"message": "User created"
}

Eventos

POST

Crear un nuevo evento.

La ruta para realizar la peticion es la siguiente:
POST https://projectncapas-c3f52370649e.herokuapp.com/API/v1/tickcheck/saveEvent

Para la creacion de un nuevo evento los parametros que debera de contener ademas del token son los siguientes:

  • name: nombre del evento
  • image: url de la imagen del evento
  • category: codigo de la categoria a la cual pertenece el evento
  • placeCode: codigo del lugar donde se desarrollara el evento.
  • eventDate: fecha cuando sera el evento
  • eventHour: hora de inicio
  • sponsor: este es una arreglo el cual contendra el nombre de los sponsors del evento.
  • ticket: arreglo de los diferentes tipos de tickets que tendra el evento, este contendra nombre, precio y cantidad
